# Data Update Guide

For human maintainers and AI agents alike :)

This guide covers how to update game data (items, ingredients, recipes, etc.) in the build-solver from the upstream [wynnbuilder-beta](https://github.com/wynnbuilder-beta/wynnbuilder-beta.github.io) source.

---

## Source of truth

All item/ingredient/recipe data originates from **wynnbuilder-beta**:

```
../wynnbuilder-beta.github.io/    ← upstream source
```

The build-solver consumes a subset of those files. Never edit the data files in build-solver directly — make changes upstream and sync them here.

---

## File map

| File | Purpose | Also copy to `dist/`? |
|---|---|---|
| `compress.json` | Items + accessories in compact form. Loaded by the frontend at runtime. | Yes |
| `clean.json` | Human-readable version of item data. Not used by the frontend, useful for debugging. | No |
| `data/<version>/` | Versioned snapshots: `items.json`, `atree.json`, `aspects.json`, `majid.json`, etc. | Yes |
| `ingreds_compress.json` | Ingredient data | No (rarely changes) |
| `ingreds_clean.json` | Human-readable ingredients | No |
| `recipes.json` / `recipes_compress.json` / `recipes_clean.json` | Crafting recipes | No (rarely changes) |
| `tomes.json` | Tome data | No (rarely changes) |

---

## How to update

### 1. Pull the latest from wynnbuilder-beta

```bash
cd ../wynnbuilder-beta.github.io
git pull
```

Check what changed:

```bash
git log --oneline -5
```

Look for commits mentioning items, atree, or a new version number (e.g. `2.2.0.14`).

### 2. Find the latest data version

```bash
ls data/
```

The newest folder name (e.g. `2.2.0.14`) is the version to sync.

### 3. Copy files into build-solver

From the repo root:

```bash
# Core item file — always update this
cp ../wynnbuilder-beta.github.io/compress.json ./compress.json
cp ./compress.json ./dist/compress.json

# Readable item file — update if you want clean.json in sync
cp ../wynnbuilder-beta.github.io/clean.json ./clean.json

# New versioned data folder — copy if it doesn't exist yet
NEW_VER=2.2.0.14   # replace with the actual new version
cp -r ../wynnbuilder-beta.github.io/data/$NEW_VER ./data/$NEW_VER
cp -r ./data/$NEW_VER ./dist/data/$NEW_VER
```

### 4. Check ingredients/recipes/tomes (if needed)

These change rarely. Verify before copying:

```bash
diff ./ingreds_compress.json ../wynnbuilder-beta.github.io/ingreds_compress.json
diff ./recipes.json           ../wynnbuilder-beta.github.io/recipes.json
diff ./tomes.json             ../wynnbuilder-beta.github.io/tomes.json
```

If they differ and the upstream changes are intentional, copy them:

```bash
cp ../wynnbuilder-beta.github.io/ingreds_compress.json ./ingreds_compress.json
cp ../wynnbuilder-beta.github.io/ingreds_clean.json    ./ingreds_clean.json
cp ../wynnbuilder-beta.github.io/recipes.json          ./recipes.json
cp ../wynnbuilder-beta.github.io/recipes_compress.json ./recipes_compress.json
cp ../wynnbuilder-beta.github.io/recipes_clean.json    ./recipes_clean.json
cp ../wynnbuilder-beta.github.io/tomes.json            ./tomes.json
```

### 5. Verify

Sanity check the copy worked:

```bash
# compress.json should match wynnbuilder-beta exactly
diff ./compress.json ../wynnbuilder-beta.github.io/compress.json
diff ./dist/compress.json ./compress.json

# New version folder should exist in both root and dist
ls ./data/ | tail -3
ls ./dist/data/ | tail -3
```

Run the dev server and confirm items load without errors:

```bash
npm run dev
```

---

## What NOT to update manually

- `dist/assets/` — this is generated by `npm run build`, do not touch it
- `workbench/data/` — separate legacy workbench data, not used by the main frontend
- `dps_data.json`, `tome_map.json`, `maploc_*.json`, `terrs_*.json`, `skillpoints.csv` — sync these only if upstream explicitly changed them

- [ ] Copied `clean.json` → root
- [ ] Copied new `data/<version>/` → root and `dist/`
- [ ] Checked ingredients/recipes/tomes for changes
- [ ] Verified `dist/compress.json` matches root `compress.json`
- [ ] Dev server loads items without errors
